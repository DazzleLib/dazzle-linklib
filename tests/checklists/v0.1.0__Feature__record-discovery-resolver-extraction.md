**Target version:** 0.1.0 (P2 extraction increment)
**Ships with commit:** TBD (pre-commit)

`test_record.py` (20), `test_discovery.py` (11), `test_resolver.py` (7),

## What this is

`dazzle-linklib` is **L2** of the DazzleLib stack: the content-addressable
link-record library extracted from the `dazzlelink` CLI tool. This increment
ports the record model (`DazzleLinkData`), the record-file discovery/rebase
surface (`find_dazzlelinks` / `scan` / `rebase`), and the injectable target
resolver (`resolve_target`). This checklist verifies the things mocks can't:
that the library reads/writes **real** `.dazzlelink` files produced by the
shipped tool, behaves correctly against the **real** filesystem (symlinks, UNC,
BOM, missing targets), and imports cleanly as a standalone package. Run it
before tagging the first P2 release.

## Prerequisites

1. **OS**: any (Windows is the primary target; the cross-tool and symlink tests
   are most meaningful on Windows).
2. **Python**: 3.9+.
3. **Install the library (editable)** from the repo root:

   **cmd.exe**:
   ```cmd
   cd C:\code\dazzle-linklib
   pip install -e . --no-deps
   ```

   **PowerShell**:
   ```powershell
   Set-Location C:\code\dazzle-linklib
   pip install -e . --no-deps
   ```

   **POSIX**:
   ```bash
   cd /c/code/dazzle-linklib && pip install -e . --no-deps
   ```

4. **Bedrock dependency present**: `dazzle-lib` must be importable
   (`python -c "import dazzle_lib"`). It is the only hard dependency right now.
5. **For the cross-tool section only**: the published `dazzlelink` tool installed
   in the same interpreter (`pip show dazzlelink`). If absent, those tests skip
   cleanly — install with `pip install dazzlelink` to run them.
6. **State isolation**: every test below writes only to a throwaway temp
   directory you create (see each section). Nothing touches real dotfiles.

### Reset between sections

Delete the scratch directory you created and remake it. Prefer `dz safedel` if
available; otherwise:

  **cmd.exe**: `rmdir /s /q %TEMP%\dll_check & mkdir %TEMP%\dll_check`
  **PowerShell**: `Remove-Item -Recurse -Force $env:TEMP\dll_check; New-Item -ItemType Directory $env:TEMP\dll_check`
  **POSIX**: `rm -rf /tmp/dll_check && mkdir /tmp/dll_check`

## High-Value Verification (~5 minutes)

If you only have five minutes, run these. If any fail, the increment is broken.

### HV.1 — The whole automated suite is green

Validates: nothing regressed across record/discovery/resolver/cross-tool.

- [ ] **Steps:** from the repo root: `python -m pytest -q`
- [ ] **Expected:** `53 passed` (5 cross-tool tests may show as skipped if the
  `dazzlelink` tool is not installed — that is acceptable).
- [ ] **Why this matters:** this is the regression floor; the detailed sections
  assume it holds.

### HV.2 — The library imports standalone (the user's acceptance test)

Validates: the package stands on its own and exposes the full P2 surface.

- [ ] **Steps:**
  ```
  python -c "import dazzle_linklib as L; print(sorted(L.__all__))"
  ```
- [ ] **Expected:** a list including `DazzleLinkData`, `find_dazzlelinks`,
  `scan`, `rebase`, `resolve_target`, `ReachabilityResolver`,
  `DazzleLinkError`, `DazzleLinkException`, `default_reachability`.
- [ ] **Why this matters:** the stated design test is "the library can be
  directly imported into dazzlelink as a standalone tool." If import fails or
  the surface is incomplete, the design is wrong.

### HV.3 — A real tool-written `.dazzlelink` reads in the library

Validates: wire compatibility with the shipped tool's output.

- [ ] **Steps** (requires `dazzlelink` installed):
  ```
  python -c "import tempfile,os; from dazzlelink.data import DazzleLinkData as T; from dazzle_linklib import DazzleLinkData as L; p=os.path.join(tempfile.mkdtemp(),'x.dazzlelink'); r=T(); r.set_original_path('A'); r.set_target_path('B'); r.save_to_file(p); print(L.from_file(p).get_target_path())"
  ```
- [ ] **Expected:** prints `B`.
- [ ] **Why this matters:** if the library can't read the tool's files, the
  extraction is not a drop-in and every existing `.dazzlelink` on disk is at risk.

### HV.4 — `resolve_target` picks the first reachable locator

Validates: the injectable resolver core behavior (AC-3).

- [ ] **Steps:**
  ```
  python -c "from dazzle_linklib import DazzleLinkData, resolve_target; r=DazzleLinkData(); r.add_locator('url','u1'); r.add_locator('ipfs','good'); print(resolve_target(r, reachability=type('F',(),{'is_reachable':lambda s,v: v=='good'})()))"
  ```
- [ ] **Expected:** prints `{'kind': 'ipfs', 'value': 'good'}`.
- [ ] **Why this matters:** Relinker depends on injecting its own reachability
  checker; the candidate-walk + first-reachable contract must hold.

### HV.5 — D3 generalization is invisible to legacy records

Validates: a path-only record gains no new keys (additive, non-breaking).

- [ ] **Steps:**
  ```
  python -c "from dazzle_linklib import DazzleLinkData; d=DazzleLinkData(); d.set_original_path('p'); k=d.to_dict(); print('content_id' in k, 'relations' in k)"
  ```
- [ ] **Expected:** prints `False False`.
- [ ] **Why this matters:** if generalization leaks keys into ordinary records,
  every consumer's golden files and diffs break.

### HV.6 — L2 imports nothing from its consumers (AC-4)

Validates: dependencies point down only — no circular import with dazzlelink.

- [ ] **Steps:** `python -m pytest tests/test_no_upstream_imports.py -q`
- [ ] **Expected:** `1 passed`.
- [ ] **Why this matters:** an upward import would make the library
  un-shippable on its own and re-couple it to the tool it was extracted from.

## Section 1: Record model on real files (~5 min)

Create a scratch dir first (see "Reset between sections").

### 1.1 Round-trip a record to disk and back

- [ ] **Steps:**
  ```
  python -c "import os,tempfile; from dazzle_linklib import DazzleLinkData; d=tempfile.mkdtemp(); p=os.path.join(d,'a.dazzlelink'); r=DazzleLinkData(); r.set_original_path(r'C:\src\f.txt'); r.set_target_path(r'D:\dst\f.txt'); r.set_default_mode('open'); print('saved', r.save_to_file(p)); q=DazzleLinkData.from_file(p); print(q.get_original_path(), q.get_target_path(), q.get_default_mode())"
  ```
- [ ] **Expected:** `saved True` then the two paths and `open`.
- [ ] **Check:** open the written file in an editor — it is pretty-printed JSON
  (2-space indent), UTF-8, with `schema_version: 1`.

### 1.2 Legacy flat-format file reads

- [ ] **Steps** (write a flat record, then read it):

  **cmd.exe**:
  ```cmd
  echo {"target_path":"D:\\old\\t.txt"} > %TEMP%\dll_check\flat.dazzlelink
  python -c "from dazzle_linklib import DazzleLinkData; print(DazzleLinkData.from_file(r'%TEMP%\dll_check\flat.dazzlelink').get_target_path())"
  ```

  **PowerShell**:
  ```powershell
  '{"target_path":"D:\\old\\t.txt"}' | Out-File -Encoding utf8 $env:TEMP\dll_check\flat.dazzlelink
  python -c "from dazzle_linklib import DazzleLinkData; import os; print(DazzleLinkData.from_file(os.path.join(os.environ['TEMP'],'dll_check','flat.dazzlelink')).get_target_path())"
  ```

  **POSIX**:
  ```bash
  printf '{"target_path":"D:\\\\old\\\\t.txt"}' > /tmp/dll_check/flat.dazzlelink
  python -c "from dazzle_linklib import DazzleLinkData; print(DazzleLinkData.from_file('/tmp/dll_check/flat.dazzlelink').get_target_path())"
  ```
- [ ] **Expected:** prints `D:\old\t.txt`.
- [ ] **Why this matters:** legacy files in the wild must still load. (Note: if
  your shell's `echo`/`Out-File` adds a BOM or CRLF, the read must still
  succeed — that is itself the BOM-tolerance test.)

### 1.3 Polyglot embedded-script file reads via exact-line marker

This is the bug-prone path: the marker literal appears twice (once as a decoy
comment, once as the real delimiter). The library must match the **exact line**.

- [ ] **Steps:**
  ```
  python -c "import os,tempfile,json; from dazzle_linklib import DazzleLinkData; d=tempfile.mkdtemp(); p=os.path.join(d,'exe.dazzlelink'); open(p,'w',encoding='utf-8').write('#!/usr/bin/env python\n# decoy: # DAZZLELINK_DATA_BEGIN in a comment\nprint(1)\n# DAZZLELINK_DATA_BEGIN\n'+json.dumps({'link':{'original_path':'EMBEDDED'}})+'\n'); print(DazzleLinkData.from_file(p).get_original_path())"
  ```
- [ ] **Expected:** prints `EMBEDDED` (NOT a parse error, NOT the decoy).
- [ ] **Why this matters:** a substring match (the original source bug) would
  grab the decoy line and fail. Real executable `.dazzlelink` files would break.

### 1.4 Corrupt file raises a typed error

- [ ] **Steps:**
  ```
  python -c "import os,tempfile; from dazzle_linklib import DazzleLinkData, DazzleLinkError; d=tempfile.mkdtemp(); p=os.path.join(d,'bad.dazzlelink'); open(p,'w').write('garbage, no json, no marker'); 
  try:
   DazzleLinkData.from_file(p); print('NO ERROR (bad)')
  except DazzleLinkError as e: print('raised DazzleLinkError OK')"
  ```
- [ ] **Expected:** `raised DazzleLinkError OK`.
- [ ] **Why this matters:** consumers catch `DazzleError`/`LinkError`; the error
  must be the stack-rooted type, not a bare `ValueError`/`KeyError`.

## Section 2: Discovery + rebase on the real filesystem (~5 min)

### 2.1 `find_dazzlelinks` and `scan` over a real tree

- [ ] **Steps:** create a dir with two records and a subdir with one more, then:
  ```
  python -c "import os,tempfile; from dazzle_linklib import DazzleLinkData, find_dazzlelinks, scan; d=tempfile.mkdtemp(); os.mkdir(os.path.join(d,'sub')); [DazzleLinkData().save_to_file(os.path.join(d,n)) for n in ('a.dazzlelink','b.dazzlelink')]; DazzleLinkData().save_to_file(os.path.join(d,'sub','c.dazzlelink')); print('flat', len(find_dazzlelinks(d))); print('recursive', len(scan(d, recursive=True)))"
  ```
- [ ] **Expected:** `flat 2` then `recursive 3`.
- [ ] **Why this matters:** discovery walks the real filesystem; mocks can't
  prove `os.walk`/glob behavior on your actual OS and path separators.

### 2.2 `rebase` recomputes a stale relative path against a real target

- [ ] **Steps:**
  ```
  python -c "import os,tempfile,json; from dazzle_linklib import DazzleLinkData, rebase; d=tempfile.mkdtemp(); t=os.path.join(d,'real.txt'); open(t,'w').write('hi'); p=os.path.join(d,'link.dazzlelink'); r=DazzleLinkData(); r.set_target_path(t); r.data['link']['target_representations']={'original_path':t,'relative_path':'WRONG/stale.txt'}; r.save_to_file(p); rep=rebase(d); print('changed', len(rep['changed']), rep['changed'][0]['action'] if rep['changed'] else ''); print('new_rel', json.load(open(p))['link']['target_representations']['relative_path'])"
  ```
- [ ] **Expected:** `changed 1 Recomputed relative from absolute`, and
  `new_rel real.txt`.
- [ ] **Why this matters:** rebase touches real file existence checks and
  writes back to disk — exactly where atomicity/encoding regressions hide.

### 2.3 `rebase` reports both-broken as an error (no crash, no write)

- [ ] **Steps:**
  ```
  python -c "import os,tempfile; from dazzle_linklib import DazzleLinkData, rebase; d=tempfile.mkdtemp(); p=os.path.join(d,'b.dazzlelink'); r=DazzleLinkData(); r.set_target_path(os.path.join(d,'gone.txt')); r.data['link']['target_representations']={'original_path':os.path.join(d,'gone.txt'),'relative_path':'also/gone.txt'}; r.save_to_file(p); rep=rebase(d); print('errors', len(rep['errors']), 'Both paths broken' in rep['errors'][0]['error'])"
  ```
- [ ] **Expected:** `errors 1 True`.

## Section 3: Resolver with a real and an injected checker (~3 min)

### 3.1 Default (filesystem) reachability picks the existing path

- [ ] **Steps:**
  ```
  python -c "import os,tempfile; from dazzle_linklib import DazzleLinkData, resolve_target; d=tempfile.mkdtemp(); real=os.path.join(d,'here.txt'); open(real,'w').write('x'); r=DazzleLinkData(); r.add_locator('path', os.path.join(d,'nope.txt')); r.add_locator('path', real); print(resolve_target(r))"
  ```
- [ ] **Expected:** prints a dict whose `value` is the existing `here.txt` path.
- [ ] **Why this matters:** the default checker must consult the real
  filesystem; this can't be proven by a mock-only run.

### 3.2 Injected checker overrides the default (Relinker pattern)

- [ ] **Steps:**
  ```
  python -c "from dazzle_linklib import DazzleLinkData, resolve_target, ReachabilityResolver; r=DazzleLinkData(); r.add_locator('url','https://b.example/y'); class Net:\n  def is_reachable(self,v): return v.startswith('https://')\nprint(isinstance(Net(), ReachabilityResolver)); print(resolve_target(r, reachability=Net()))"
  ```
- [ ] **Expected:** `True` then `{'kind': 'url', 'value': 'https://b.example/y'}`.
- [ ] **Why this matters:** a structural Protocol must accept a fake with no base
  class and no stack import — the injection seam Relinker relies on.

## Section 4: Cross-tool bidirectional compatibility (~3 min, needs `dazzlelink`)

Skip if `pip show dazzlelink` reports nothing.

### 4.1 Library-written file reads in the tool

- [ ] **Steps:**
  ```
  python -c "import os,tempfile; from dazzle_linklib import DazzleLinkData as L; from dazzlelink.data import DazzleLinkData as T; p=os.path.join(tempfile.mkdtemp(),'l.dazzlelink'); r=L(); r.set_original_path('O'); r.set_target_path('G'); r.save_to_file(p); print(T.from_file(p).get_target_path())"
  ```
- [ ] **Expected:** prints `G`.

### 4.2 L2 extensions survive a tool round-trip

- [ ] **Steps:**
  ```
  python -c "import os,tempfile; from dazzle_linklib import DazzleLinkData as L; from dazzlelink.data import DazzleLinkData as T; d=tempfile.mkdtemp(); p1=os.path.join(d,'1.dazzlelink'); p2=os.path.join(d,'2.dazzlelink'); r=L(); r.add_locator('ipfs','QmX'); r.set_content_id('sha256','beef'); r.save_to_file(p1); T.from_file(p1).save_to_file(p2); back=L.from_file(p2); print(back.get_content_id())"
  ```
- [ ] **Expected:** prints `{'algorithm': 'sha256', 'digest': 'beef'}`.
- [ ] **Why this matters:** the tool doesn't understand `content_id`, but a
  generic JSON round-trip through it must not strip the new keys.

## What the automated tests DON'T cover

- **Real `.dazzlelink` files in the wild** — files produced by older tool
  versions, with real UNC/`subst`/drive paths, mixed line endings, or BOMs from
  real editors. The tests synthesize fixtures; only manual runs hit real data.
- **Symlink/junction behavior on Windows** — `rebase` and resolution against
  actual NTFS junctions/symlinks (creation needs elevation or Developer Mode).
- **Encoding/atomicity under stress** — rebase writing to a read-only file,
  disk-full, or antivirus locking the file mid-write.
- **The real installed tool** — CI for this library won't have `dazzlelink`
  installed, so the cross-tool section only runs where a human installs both.
- **Performance feel** — does `scan`/`find_dazzlelinks` over a large real tree
  feel snappy? Not asserted anywhere.
- **Error message clarity** — is the `DazzleLinkError` text actually helpful when
  a real corrupt file is hit?
